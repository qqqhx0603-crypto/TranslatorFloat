# 模型列表与预制模型

## 2026-05-01

- 主模型下拉框只显示当前本地 `API.txt` 中保存的模型；如果没有模型，则使用内置推荐模型临时兜底。
- “预制模型”按钮会调用硅基流动 `/v1/models?sub_type=chat` 获取最新聊天模型列表。
- 预制模型弹窗按内置速度偏好排序：`tencent/Hunyuan-MT-7B`、`inclusionAI/Ling-flash-2.0`、`deepseek-ai/DeepSeek-V3.2` 等优先，其余保持接口返回顺序。
- 弹窗默认勾选当前主下拉框里可选的模型；保存后只写回对应版本自己的 `API.txt`，不清空缓存。

## 2026-05-09

- 修复模型级 UI 缓存：每个模型分别保存 `快速/思考`、上下文档位、推理强度。
- 切到不支持思考的模型时可以临时切回快速，但切回支持思考的模型会恢复该模型上次保存的思考设置。
- 翻译结果缓存 key 已确认包含模型、模式、镜像配置、推理强度、上下文策略、提示词和请求参数；快速/思考不会共用同一条翻译缓存。

## 2026-05-10

- Python/EXE 版“预制模型”弹窗改为每行一个带模型名的勾选控件，勾选框在左、模型名紧随其后。
- PowerShell 版使用 `CheckedListBox`，系统控件本身就是左侧勾选框，无需改动。
- 2026-06-04 修复 PowerShell 版模型列表污染：模型数组函数不能用 `return ,@(...)` 形成嵌套数组，否则 WinForms 下拉框会把整个 `string[]` 转成空格拼接的一条模型 ID。
- 2026-06-04 PowerShell 版新增模型 ID 校验：含空白的模型字符串视为损坏数据，读取 `selectedModel`、`thinkingSupport`、`modelSettings`、API 返回模型和保存当前模型设置时都会跳过。
